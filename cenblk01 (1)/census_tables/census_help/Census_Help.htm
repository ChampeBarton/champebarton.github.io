Using the 2000 Census Data

The 2000 Census has produced an enormous body of data. Originally, these data have been released as 39 tables (in Summary File 1) each consisting of upwards of two hundred attribute fields. In order to condense this information and distribute it through The Florida Geographic Data Library (FGDL), we've selected and processed 5 tables that seemingly contain the most important and commonly used attributes. These 5 tables combined have over 1000 attribute fields that include demographic data on total population, race, sex, age, and household.

The following help topics are provided to help you make use of this large and important data set. They outline methods for joining census tables and extracting different geographic boundaries (e.g., tracts, TAZs, counties, etc.).

How to Link Tables (ArcGIS 8.1)
In order to link the supplemental census tables (sf1##.dbf - sf4##.dbf, sf37##.dbf) to the census shapefile (cenblk##.shp) in ArcGIS 8.1, use the following procedure:

  1. Determine what attribute information that you want to link (e.g. age, sex, etc.). In order to determine what attribute data you are looking for refer to the metadata section labeled "FEATURE ATTRIBUTES TABLES CODES AND VALUES". Scroll down to the supplemental tables and find the table that has the attribute items of interest. Take note of the table number (e.g., sf1, sf2, sf3, sf4, sf37) and the item name (e.g. P001001) for the attribute (e.g., Total).
  2. Start ArcGIS. For the county of interest, add cenblk##.shp.
  3. Right Click on cenblk##.shp. Choose Joins and Relates... > Join.
  4. "What do you want to join to this layer" should be "Join attributes from a table" (Fig. 1). Choice 1 should be "LOGRECNO". Choice 2 should be the table chosen from step 1 and Choice 3 should be "LOGRECNO". Click "OK".

How to extract Block Groups, Tracts, TAZs, ZCTAs, and Counties (ArcGIS 8.1)
The census data in the CENBLK## shapefile are at the block level, the smallest unit of census data. Using GIS techniques, it is possible to summarize the data by Block Group, Tract, Traffic Analysis Zone (TAZ), Zip Code Tabulation Area (ZCTA), or County. Outlined below, are two different methods for extracting census data for different geographic boundaries.

Method 1: Using the Layer Properties (Visual Representation Only)

  1. Right Click on cenblk##.shp select Properties > Symbology Tab
  2. Select Categories > Unique values and set the Value Field to the Census Entity that you wish to show (Fig. 2). Use the following value fields for the corresponding Census Entity.
     
    Census Entity Value Field
    Block Group "Bggrp_disv"
    Tract "Tract_disv"
    Traffic Analysis Zone (TAZ) "Taz_disv"
    ZCTA3 "Zcta3"
    ZCTA5 "Zcta5"
    County "County"


    The figure below illustrates the procedure for showing Census Tracts. Note that you need to remove the outline of each symbol before pressing "OK" (Right click on a value and select "Properties for all symbols..." and set the outline color to no color).

Method 2: Performing Dissolve also called "summarizing" or "aggregating" (Visual and Physical Change of Data)

  1. Open ArcMAP 8.1 and add cenblk##.shp to a view.
  2. If you want information on a specific attribute then perform the steps outlined in How To Link Tables (ArcGIS 8.1)
  3. Select Tools > GeoProcessing Wizard...
  4. Select "dissolve features based on an attribute" and click "Next"
  5. "Select the theme to dissolve:" as the cenblk##.shp (Fig. 3).
  6. "Select an attribute to dissolve" should be based on what Census Entity you want to extract. Use the following attribute fields for the corresponding Census Entity.
     
    Census Entity Attribute to Dissolve
    Block Group "Bggrp_disv"
    Tract "Tract_disv"
    Traffic Analysis Zone (TAZ) "Taz_disv"
    ZCTA3 "Zcta3"
    ZCTA5 "Zcta5"
    County "County"

    The figure below illustrates the procedure for extracting Census Tracts. Specify where the output file should go and press "Next".


    Figure 3. Dissolving Features by Attribute with Geoprocessing Wizard



  7. "Choose one or more additional fields or operations to be included in the output shapefile or feature class." This option should be set to the individual field or fields for which you would like to get attribute data (Fig. 4). For example, if you wanted total population (P001001) for the Census Entity you would select "P001001" and check "Sum". By selecting "Sum" the individual blocks that make up the Census Entity will be added together and represented in the output shapefile. Once all the wanted attribute fields are selected click "Finish".


    Figure 4. Choose additional fields to be included in output shapefile

  4. Click on the field labeled "Logrecno" in the sf table.
  5. Click on the field labeled "Logrecno" in cenblk##.shp.
  6. With the cenblk## table active, click the "join button" . The cenblk##.shp should now contain the attribute table with the appropriate attribute.
